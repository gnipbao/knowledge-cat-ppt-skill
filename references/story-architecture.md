# Story Architecture

Use this file to turn source material into a deck brief, narrative spine, action titles, and a slide plan.

## Core Premise

A deck is a guided audience shift. The audience starts with one belief, question, or uncertainty and should leave with a clearer decision, action, or model.

## Narrative Spine Patterns

### Executive Recommendation

Use for strategy, consulting, board, business, and operating reviews.

```md
Context -> tension -> diagnosis -> options -> recommendation -> next steps
```

### Investor Or Sales Pitch

Use when the deck must create belief and urgency.

```md
Problem -> why now -> solution -> proof -> market or value -> business model -> ask
```

### Teaching Or Course Deck

Use when the user wants comprehension and recall.

```md
Promise -> mental model -> worked example -> practice -> pitfalls -> recap
```

### Academic Or Research Talk

Use for conference talks, defenses, seminars, and research briefings.

```md
Question -> gap -> method -> finding -> interpretation -> limitation -> conclusion
```

### Product Or Technical Demo

Use when explaining a product, architecture, workflow, or agent.

```md
Old workflow -> bottleneck -> new system -> how it works -> proof -> rollout
```

### Social Carousel

Use for Xiaohongshu, LinkedIn, WeChat cards, or short-form educational content.

```md
Hook -> tension -> framework -> steps -> example -> save/share prompt
```

## Action Titles

An action title states the slide's point.

Weak:

- Market Overview
- Product Features
- Roadmap
- Results

Strong:

- The market is shifting from generic automation to role-specific agents
- Our current funnel loses trust before the demo call
- The new workflow removes two handoffs and cuts review time in half
- Three signals suggest the category is ready for a paid entry product

Rules:

- Prefer a complete sentence.
- Make the claim specific enough to be debated.
- Avoid cleverness that obscures the point.
- Do not repeat the deck title as a slide title.
- Use topic labels only while drafting, then convert them.

## Ghost Deck Test

Read only the action titles in order. They should tell the story without body text.

Pass criteria:

- The sequence has a beginning, middle, and end.
- Each title advances the argument.
- No two slides make the same point.
- The final title creates a decision, conclusion, or next action.

## Slide Role Library

Use these roles to avoid random slide piles:

- Hook: earns attention.
- Context: frames the situation.
- Tension: names the problem or gap.
- Diagnosis: explains root cause.
- Mechanism: shows how the solution works.
- Proof: provides evidence.
- Comparison: contrasts options or before/after.
- Case: makes the abstract concrete.
- Process: explains steps.
- System map: shows relationships.
- Data exhibit: turns numbers into a point.
- Decision: asks for a choice.
- Roadmap: sequences execution.
- Recap: compresses memory.
- Appendix: stores detail without breaking the main story.

## Evidence Mapping

Each substantive slide needs an evidence type:

- User-provided data
- Public source
- Direct quote
- Internal observation
- Model output or inference
- Illustrative example
- Assumption

Label assumptions. Do not present inference as fact.

## Plan Quality Checks

- The deck has one primary audience.
- The deck has one primary outcome.
- The first three slides create context and stakes.
- The middle slides contain proof, not only claims.
- The ending asks for action, conclusion, or memory.
- Every slide has a role.
- Every evidence slide has a source, assumption, or caveat.
- There is no unbroken run of text-only slides.
- The planned visuals are feasible in the selected output lane.
