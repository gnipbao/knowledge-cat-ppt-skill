# Design Systems

Use this file when selecting a visual direction, layouts, image treatment, or theme for a deck.

## Design Principle

Design should make the argument easier to understand. Do not decorate uncertainty. Use visual contrast, space, hierarchy, and evidence treatment to guide attention.

## Knowledge Cat Style Directions

Choose one primary direction. Do not mix them without a reason.

| Direction | Best For | Visual Language |
|---|---|---|
| Strategic Minimal | executive, consulting, strategy, investor | white or neutral canvas, strong grid, restrained accent, crisp charts |
| Editorial Proof | thought leadership, essays, lectures | magazine rhythm, strong headlines, pull quotes, photography, narrow text columns |
| Data Command | analytics, reports, market maps | dashboards, annotated charts, big numbers, tables only in appendix |
| Academic Clarity | research, thesis, seminars | white background, action titles, one exhibit per slide, citations |
| Product Lab | product, SaaS, AI tools, demos | UI screenshots, system diagrams, feature flows, clean tech palette |
| Teaching Board | courses, workshops, explainers | progressive disclosure, diagrams, examples, recap pages |
| Social Carousel | Xiaohongshu, LinkedIn, WeChat | large hook text, repeated visual rhythm, saveable frameworks |
| Vision Keynote | public talk, launch, culture | bold pacing, hero pages, high contrast, fewer words |

## Layout Rhythm

Build rhythm across the deck:

- Start with a hook or framing slide.
- Alternate dense proof slides with spacious synthesis slides.
- Use a visual reset every 3 to 5 slides for long decks.
- Avoid three consecutive slides with the same structure.
- Use appendix for detail that would slow the main story.

## Layout Families

Use a named layout family per slide:

- Full-bleed statement
- Title plus evidence exhibit
- Two-column argument
- Before/after comparison
- Timeline
- Process ladder
- System map
- Data dashboard
- Case card
- Image plus annotation
- Quote plus implication
- Decision matrix
- Closing recap

## Typography

Rules:

- Slide titles should dominate body text.
- Body text should be readable from the intended room or screen.
- Use one primary sans-serif family unless the style direction justifies a second display face.
- Use weight and scale before adding color.
- Avoid tiny footnotes except for citations.
- Do not center long body text.

## Color

Rules:

- Use a neutral base plus one or two accents for most professional decks.
- Choose colors from the topic, brand, audience, or evidence type.
- Avoid generic purple-blue gradients as a default.
- Avoid low-contrast gray text on light backgrounds.
- Use color to encode meaning, not to fill space.
- Keep the palette consistent across charts, diagrams, and callouts.

## Imagery

Rules:

- Images should carry evidence, context, emotion, or product specificity.
- Avoid generic stock photos when the user needs trust or inspection.
- Match image ratio to the layout slot before generating or cropping.
- Do not crop faces, UI details, chart labels, or product surfaces accidentally.
- Do not place generated image text where exact wording matters unless the model can reliably render it.
- Use screenshots as evidence. If beautifying screenshots, preserve information unless the user asks for redesign.

## Charts And Data

Rules:

- One chart should answer one question.
- Annotate the point directly on the chart when possible.
- Use tables only when comparison at exact values matters.
- Remove gridlines, legends, and labels that do not support the claim.
- Keep chart colors consistent with the deck color semantics.
- Cite data source or label it as user-provided.

## Motion

Use motion only when it helps attention or sequencing.

Good uses:

- Step-by-step reveal
- Transition between sections
- Demonstrating flow or causality
- Reducing cognitive load in a complex diagram

Bad uses:

- Constant decoration
- Motion behind dense text
- Long animations before every slide
- Animation that breaks PDF/PPT export needs

## Anti-Generic Rules

Avoid:

- Title plus six bullets as the default pattern
- Random icons without informational value
- Decorative gradient blobs or orbs
- Over-rounded floating cards everywhere
- Thin gray text on off-white backgrounds
- Stock-like blurry hero images
- Repeating the same layout across most slides
- Inconsistent margins and gaps
- Accent lines under every title
- Screenshots inside fake laptop frames unless it serves the story

## Accessibility And Readability

Check:

- Contrast is strong enough.
- Text is not too small.
- Important labels are readable.
- Charts do not rely only on color.
- Speaker notes or handouts cover detail removed from slides.
- Export path preserves the intended reading experience.
